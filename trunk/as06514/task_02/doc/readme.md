## Task 2. Unit testing

### Цель работы
Написание модульных тестов для программы, разработанной в лабораторной работе №1, с использованием фреймворка Google Test и анализ покрытия кода.

### Исходные данные
Из лабораторной работы №1 имеем математические модели объекта управления:

**Линейная модель:**
y_{τ+1} = ay_τ + bu_τ

**Нелинейная модель:**
y_{τ+1} = ay_τ - by_{τ-1}² + cu_τ + dsin(u_{τ-1})


где:
- `y_τ` - температура в момент времени τ
- `u_τ` - управляющее воздействие
- `a, b, c, d` - коэффициенты моделей

### Выполнение работы

#### Теоретическая часть

Модульное тестирование - это процесс проверки корректности работы отдельных модулей программы. Для реализации тестов использован фреймворк Google Test, который предоставляет:

- Макросы для создания тестов (`TEST`, `TEST_F`)
- Различные виды проверок (`EXPECT_EQ`, `EXPECT_NEAR`, `EXPECT_GT` и др.)
- Автоматическое обнаружение и запуск тестов
- Детальные отчеты о результатах тестирования

##### Реализация тестов

Написано 6 модульных тестов, покрывающих основные аспекты работы программы:

1. **LinearModelTest.BasicCalculation** - проверка корректности вычислений линейной модели
2. **LinearModelTest.ZeroInput** - проверка граничного случая с нулевыми входами
3. **NonlinearModelTest.BasicCalculation** - проверка корректности вычислений нелинейной модели
4. **NonlinearModelTest.ZeroInputs** - проверка граничного случая с нулевыми входами
5. **ConstantsTest.ValuesInRange** - проверка допустимости значений параметров моделей
6. **StepResponseTest.Convergence** - проверка сходимости линейной системы
